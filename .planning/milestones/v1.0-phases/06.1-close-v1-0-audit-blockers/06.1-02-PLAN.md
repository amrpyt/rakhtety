# Plan 06.1-02: Server-Safe Dashboard And Admin APIs

## Objective

Move sensitive dashboard and employee flows onto authenticated server APIs.

## Tasks

1. Make `/api/dashboard/summary` use the request-bound Supabase server client for all data.
2. Add `PATCH /api/employees/[id]` with `manageEmployees`.
3. Add `DELETE /api/employees/[id]` with `manageEmployees`.
4. Update employee hooks to use the API for edit, delete, deactivate, and reactivate.
5. Verify lint, tests, typecheck, and build.

## Done Means

- Dashboard summary no longer calls browser-wired repositories inside a server route.
- Employee admin mutations no longer call browser-side service methods.
