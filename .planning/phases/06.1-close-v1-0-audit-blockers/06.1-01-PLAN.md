# Plan 06.1-01: Atomic Client Intake

## Objective

Make new client creation and mandatory intake documents travel together in one request.

## Tasks

1. Update the browser client create path to send multipart form data when files exist.
2. Keep server-side rollback behavior if storage or metadata insert fails.
3. Keep JSON create support for existing non-file callers.
4. Verify lint, tests, typecheck, and build.

## Done Means

- The old client-first-then-upload flow is gone from create client.
- Required files are validated before request send and again on the server.
- A failed document upload does not leave a valid saved client.
