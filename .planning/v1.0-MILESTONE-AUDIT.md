## Closed Cross-Phase Issues

1. **Client intake is not atomic**
   - Affected: CRM-01, CRM-04
   - Status: CLOSED
   - Evidence: `src/lib/client-data/directory-client.ts`, `src/app/api/clients/route.ts`
   - Plain English: the browser now sends files with client creation; the server rolls back files and client on failure.

2. **Dashboard API uses the wrong Supabase path**
   - Affected: DASH-01..05, FIN-05, EMP-03
   - Status: CLOSED
   - Evidence: `src/app/api/dashboard/summary/route.ts`, `src/lib/services/dashboard-summary.service.ts`, `src/lib/database/repositories/workflow.repository.ts`
   - Plain English: the server route now passes its request-bound Supabase client into the summary query path.

3. **Employee edit/delete is not fully server-gated**
   - Affected: EMP-01, AUTH-04
   - Status: CLOSED
   - Evidence: `src/hooks/useEmployees.ts`, `src/lib/client-data/directory-client.ts`, `src/app/api/employees/[id]/route.ts`
   - Plain English: edit/delete/deactivate/reactivate now go through server APIs with `manageEmployees`.

4. **Payment recording permission gate is inconsistent**
   - Affected: FIN-03
   - Status: ACCEPTED
   - Evidence: `src/hooks/useFinancials.ts`, `src/lib/services/financial.service.ts`, `src/lib/database/repositories/financial.repository.ts`
   - Plain English: RLS and existing service validation protect this v1 flow; moving it to a dedicated API is future hardening, not a v1 blocker.
