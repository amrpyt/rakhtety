# رخصتي (Rakhtety) — نظام إدارة تراخيص المكاتب الهندسية

## What This Is

منصة SaaS/ERP لإدارة سير عمل تراخيص البناء وتصاريح الحفر للمكاتب الهندسية في مصر. النظام يتتبع ملفات العملاء عبر مسارين رئيسيين (رخصة الجهاز وتصريح الحفر)، يدير المدفوعات الحكومية (رسوم) والأتعاب (ربحية المكتب)، ويوفر تقارير احترافية للعميل. حالياً للاستخدام الداخلي، ثم تغليفه كمنتج SaaS للمكاتب الهندسية الأخرى.

## Core Value

**الهدف الأساسي:** القضاء على العشوائية في إدارة التراخيص — تقليل الاعتماد على Excel، تقليل الأخطاء البشرية، توفير تكلفة العمالة، وتسريع إنجاز التراخيص.

## Context

### المستخدمون
- **المدير:** متابعة حالة الملفات والبانكنيكات
- **الموظفون:** تنفيذ الخطوات المحددة لكل ملف
- **العملاء:** استلام تقارير حالة دورية

### المتطلبات الأساسية من الموجز
1. **التحقق الصارم من المدخلات:** لا يمكن حفظ ملف جديد بدون 5 مرفقات إلزامية:
   - صورة محضر الاستلام
   - صورة البطاقة
   - توكيل
   - شهادة هندسية
   - كشف قياس
   (مرفقات اختيارية: الإيصال في حالات التعلية)

2. **الاعتمادية في مسارات العمل:**
   - لا يمكن البدء في تصريح الحفر إلا بعد استكمال رخصة الجهاز بالكامل

3. **مسار رخصة الجهاز (5 خطوات):**
   - بيان الصلاحية
   - تقديم المجمعة العشرية للإسكان المميز
   - تقديم الملف
   - دفع إذن الرخصة وشراء عقد مخلفات
   - استلام الرخصة

4. **مسار تصريح الحفر (متشعب):**
   - تقديم واستلام شهادة الإشراف
   - تقديم واستلام التأمينات
   - التقديم على العداد الإنشائي
   - تقديم ودفع واستلام تصريح الحفر
   - تصريح التعدين (الدفع والاستلام ومتابعة معاينة الجيش)

5. **الإدارة المالية:**
   - كل خطوة ترتبط بـ Fees (رسوم حكومية) و Profit (أتعاب المكتب)
   - تتبع: إجمالي التكلفة، المدفوعات، المديونية المتبقية

6. **لوحة تحكم الإدارة:**
   - معرفة حالة كل ملف وأين يتوقف (Bottlenecks)
   - تحديد الموظف المسؤول عن التأخير

7. **تقارير العميل:**
   - تقرير برنت احترافي يوضح:
     - الخطوات المنجزة (عليها علامة ✓)
     - الخطوات المتبقية
     - المبالغ المدفوعة
     - المديونية المتبقية
     - الأتعاب

### مثال ديمو موجود
`rakhtety-erp-demo.html` — واجهة HTML/CSS/JS كاملة بتصميم Nexus + Arabic-friendly

## Constraints

- **Serverless Architecture:** استخدام خدمات بـ generous free tiers
  - Frontend: Static hosting (GitHub Pages, Cloudflare Pages)
  - Database: Firebase Firestore / Supabase (free tier)
  - Auth: Firebase Auth / Supabase Auth (free tier)
  - File Storage: Firebase Storage / Supabase Storage
  - Serverless Functions: Vercel (free tier) / Cloudflare Workers (free tier)
  - Email: SendGrid (free tier) / Resend (free tier)
- **تصميم عربي:** RTL, Cairo font, Nexus design tokens
- **بدون Excel:** النظام يحل محل جميع ملفات Excel الحالية
